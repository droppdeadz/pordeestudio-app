# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Portfolio website for **Pordee Studio** — a video production studio in Chiang Mai, Thailand. Replaces their Wix site with a fast, cinematic, bilingual (TH/EN) platform. Static-first Astro site with Sanity CMS content, deployed to Cloudflare Pages.

## Tech Stack

Astro 5 (static/islands) · Tailwind CSS v4 · Sanity CMS (GROQ only) · Cloudflare Pages · TypeScript (strict) · Astro i18n (`/en/`, `/th/`) · YouTube embeds · Astro View Transitions · `@astrojs/sitemap`

## Commands

```bash
npm run dev          # Astro dev server
npm run build        # Production build (static output to dist/)
npm run preview      # Preview production build
npm run sanity       # Sanity Studio locally
```

## Architecture

### Routing & i18n
All pages live under `src/pages/[locale]/` — Astro's built-in i18n creates `/en/` and `/th/` prefixed routes. Static UI strings go in `src/i18n/{locale}.json`; CMS content uses bilingual fields `{ en: string, th: string }` selected by current locale.

### Data Flow
Sanity CMS → GROQ queries in `src/lib/sanity.ts` → fetched in Astro frontmatter (server-side at build) → rendered as static HTML. No client-side data fetching. Images served via `@sanity/image-url` builder with responsive transforms.

### Layout System
`BaseLayout.astro` wraps every page — owns `<html>`, `<head>`, View Transitions, fonts, SEO meta, and scroll animation observer. Props: `title`, `description`, `locale`, `ogImage?`, `ogType?`, `jsonLd?`. Pages extract `const { locale } = Astro.params` and pass it down. React islands (`client:load`/`client:visible`) only when JS interactivity is genuinely needed.

### SEO
`SEO.astro` component (rendered inside `BaseLayout` `<head>`) handles OG tags, Twitter cards, canonical URLs, `hreflang` alternates, and JSON-LD. Pages pass structured data via the `jsonLd` prop: `WebSite` on home, `VideoObject` on project detail, `BlogPosting` on blog detail. Sitemap auto-generated by `@astrojs/sitemap`. `robots.txt` served from `src/pages/robots.txt.ts`.

### Scroll Animations
Global `IntersectionObserver` in `BaseLayout` targets `[data-animate]` elements, adding `.is-visible` class to trigger `fade-up` keyframes defined in `globals.css`. Must re-init on `astro:after-swap` for View Transitions. Use `data-animate-stagger` on parent grids for staggered child reveals.

### Content Model (Sanity)
Three document types: **Project** (portfolio work), **BlogPost**, **SiteSettings** (singleton). All user-facing text fields are bilingual objects. Slugs auto-generate from English title.

### Styling Approach
Dark cinematic theme by default (black/near-black backgrounds). Tailwind v4 configured via CSS `@theme`, not `tailwind.config.js`. Mobile-first responsive. Thai-supporting fonts alongside main sans-serif.

## Gotchas

- **Zsh bracket escaping**: File paths with `[locale]` must be single-quoted in shell commands (e.g., `git add 'src/pages/[locale]/about.astro'`) or zsh interprets brackets as glob patterns.
- **View Transitions re-init**: Any `<script>` with DOM event listeners must also listen for `astro:after-swap` to re-attach after SPA-style navigation. See `MobileMenu.astro`, `YearFilter.astro`, `ContactForm.astro` for the pattern.
- **404 page locale**: `src/pages/404.astro` cannot use `Astro.params.locale` (it's outside `[locale]/`). It detects locale by checking if the URL path starts with `/th`.
- **`getSiteSettings()` double fetch**: Called in both `BaseLayout` (for Footer) and some pages. This is fine at build time (static) but worth noting.
- **Contact form backend**: Form validation is client-side only. Server-side email sending (Cloudflare Workers + Resend) is not yet implemented.
