# i18n Rules

- Two locales: `en` (default) and `th`.
- Static UI strings go in `src/i18n/en.json` and `src/i18n/th.json`.
- Every key added to one locale file must exist in the other.
- CMS content uses bilingual fields — select the right variant using `content.title[locale]` pattern.
- The language toggle must preserve the current page path, only swapping the locale segment.
- Set `<html lang={locale}>` on every page.
- Default locale detection: check browser locale, fallback to `en`.
- Never hardcode English or Thai text in components — always use the translation system.
